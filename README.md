# Example Quiz Funnel ظ¤ Retirement Readiness Quiz

A 12-step retirement readiness quiz funnel demo. White colorscheme with blue (`#2563eb`) and charcoal (`#1f2937`) accents, built for an older audience.

**Flow:**
1. **`/`**: 12-question quiz (9 retirement questions + first name + email + phone, each on its own step).
2. **`/results`**: A personalized retirement-readiness page generated by Claude using the user's answers and first name. Big "Book a Call With an Advisor" CTA at the top.
3. **`/book`**: Calendly embed (prefilled with name + email) for the final booking step.

Submissions are POSTed to a GoHighLevel webhook the moment the user finishes the quiz, so leads land in a CRM even if they don't click through to book.

---

## Setup

1. **Install dependencies**

   ```powershell
   npm install
   ```

2. **Configure environment variables**

   Copy the example file and fill in the values:

   ```powershell
   Copy-Item .env.local.example .env.local
   notepad .env.local
   ```

   | Variable | Required | What it's for |
   | --- | --- | --- |
   | `ANTHROPIC_API_KEY` | Yes | Powers the personalized AI analysis on `/results`. Get one at [console.anthropic.com](https://console.anthropic.com/). Without it the page still renders, using a smart template fallback. |
   | `GHL_WEBHOOK_URL` | Yes | Inbound webhook URL from GoHighLevel where leads + answers are POSTed. |
   | `NEXT_PUBLIC_CALENDLY_URL` | Yes (for the embed) | Your Calendly event URL (e.g. `https://calendly.com/your-handle/30min`). Until you set this, `/book` shows a "not configured" notice instead of a broken embed. |

3. **Run the dev server**

   ```powershell
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000).

4. **Build for production**

   ```powershell
   npm run build
   npm start
   ```

## How the GHL webhook payload looks

The `/api/submit` route POSTs JSON to `GHL_WEBHOOK_URL` like this:

```json
{
  "source": "Retirement Readiness Quiz",
  "submittedAt": "2026-05-15T12:34:56.000Z",
  "firstName": "Jane",
  "email": "jane@example.com",
  "phone": "5555550100",
  "readiness": { "score": 24, "max": 36, "pct": 67, "bucket": "Mostly On Track" },
  "answers": { "What is your current age?": "61 ظô 65", "...": "..." },
  "rawAnswers": { "currentAge": "61_65", "...": "..." }
}
```

In your GHL inbound webhook step, map these fields onto a contact:

- `firstName` ظْ First Name
- `email` ظْ Email
- `phone` ظْ Phone
- `readiness.bucket` and `readiness.pct` ظْ Custom fields (great for filtering/segmentation)
- `answers` (the humanized labels) ظْ A long-text custom field if you want the full quiz on the contact record

## Notes

- The quiz auto-saves to `localStorage` so a refresh mid-quiz doesn't lose progress.
- Final answers move to `sessionStorage` after submission, then `/results` reads them and calls `/api/analyze`.
- The AI prompt is locked down: no dollar projections, no product recommendations, no guarantees. Claude is asked only to deliver warm, plain-spoken commentary on the user's answers and frame the advisor call as the next step. Edit the `system` prompt in [app/api/analyze/route.js](app/api/analyze/route.js) if you want to change tone or rules.
- Compliance copy on `/results` notes that the page is not financial advice. Adjust to match what an advisor's compliance team requires.
